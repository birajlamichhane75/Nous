● Clean. Here's how the full pipeline now flows end-to-end:

  POST /api/cross-question
    { step: { title, prompt, type, subject? } }
         │
         ▼
  generateCrossQuestion()         ← lib/rag/generate.ts
    ├── retrieveOne(step)         ← embeds query, scores all docs, returns top-1
    │     └── cosineSimilarity()  ← single pass, no sort
    │
    ├── buildSystemPrompt(best)   ← injects RAG block with distractor seed
    │
    ├── Gemini gemini-2.0-flash   ← structured JSON output, temp 0.4
    │     responseSchema enforces exactly 4 options, one of each error type
    │
    └── shuffleAndLabel()         ← Fisher-Yates, assigns a/b/c/d
         │
         ▼
    { question, options[4], retrievedDocId }

  The key prompt constraint: Gemini is told exactly which error-type slot is
  already covered by the retrieved distractor seed and instructed not to invent
  a different wrong reasoning for it. The other three slots (including correct)
  are generated from first principles against the actual step content.
  retrievedDocId comes back in the response so you can log which misconception
  pattern was used — useful for tuning the knowledge base later.
